# 敦煌 3DShow 制作花絮（10 分钟讲解稿）

---

## 一、为什么选择敦煌

敦煌莫高窟，七百三十五个洞窟，四万五千平方米壁画，两千四百余尊彩塑，跨越北凉到元代一千六百年的营建史。它是世界上现存规模最大、内容最丰富的佛教艺术地，被誉为"墙壁上的博物馆"。

但现实是残酷的。莫高窟地处戈壁深处，每年能亲临现场的人有限。即使到了现场，洞窟内光照极暗，每批游客限时几分钟，很多洞窟因保护需要已经永久关闭。那些精美的飞天、经变画、供养人像，正在时间和沙尘中缓慢褪色。

我一直在想，如果有一种方式，能让人不受时间、距离、光照的限制，独自安静地看完每一个洞窟，想看多久就看多久，该有多好。

Web3D 给了我这个可能。浏览器是地球上装机量最大的运行环境，打开网址就能运行三维场景，不需要安装任何软件，不挑操作系统，手机电脑都能用。于是我决定用 Three.js 和 WebGL 把莫高窟搬进浏览器，让千年壁画永不褪色。

## 二、制作全过程

### 2.1 技术选型与参考

三维渲染选用 Three.js，这是目前最成熟的 Web3D 框架，底层基于 WebGL，支持 PBR 物理材质、阴影、后处理等现代渲染特性。建模选用 Blender，因为它有完整的 Python API，可以程序化生成几何体，避免大量手工操作。模型格式选用 glTF 的二进制变体 GLB，这是 Web3D 领域的事实标准，Three.js 原生支持。压缩选用 Meshopt，专为 glTF 设计的网格压缩算法，有浏览器端 WASM 解码器，可以在加载时实时解压。

配色方面，我参考了敦煌研究院发布的壁画颜料分析报告。莫高窟壁画使用的是矿物颜料，石青、石绿来自孔雀石和蓝铜矿，朱砂来自辰砂，土黄和赭石来自铁氧化物，加上金箔装饰。这些颜色的特点是饱和度低、稳定性强，历经千年仍然可辨。我将这五种矿物色的色值提取出来，构建了一套低饱和暖棕配色体系，应用到所有材质和界面设计中。

### 2.2 Blender 程序化建模

场馆建模是整个项目的基础。我用 Blender 的 Python API 编写了三千四百多行脚本，程序化生成整个展馆。

展馆是一个七十二米长、十六米宽的矩形空间，沿 X 轴中轴线排列五间洞窟。每间洞窟纵深八米、宽度六米、高度四米五，与莫高窟真实洞窟的空间比例接近。

壁画纹理也是程序化生成的。我定义了六种图案类型，分别是飞天纹、伎乐纹、经变画纹、供养人纹、藻井纹和千佛纹。每种图案配合四个朝代的参数变体，生成视觉差异化的纹理。比如西魏飞天面容清瘦、飘带细长，盛唐飞天面相丰腴、飘带华丽，晚唐飞天趋于简约，元代密宗像则线条刚硬。

彩塑人像用几何体组合方式生成，基座、躯干、头光、背光分层叠加，再根据朝代调整比例和色彩。

整个建模过程完全由脚本驱动，参数可调、结果可复现，不会因为手工操作而产生不一致。

### 2.3 模型压缩与导出

建完的模型从 Blender 导出为 GLB 文件。原始体积较大，直接放在网页上加载很慢。我使用 Meshopt 压缩算法对网格数据进行编码压缩，浏览器端通过 MeshoptDecoder 的 WASM 模块实时解码。这样模型文件体积显著缩小，而解码在加载时自动完成，对用户完全透明。

### 2.4 六层分层光照

光照是整个项目调试时间最长的环节。洞窟必须暗才有博物馆氛围，但太暗就看不清壁画和彩塑的细节。我反复调试后设计了六层分层光照系统。

第一层是环境光，暖棕低饱和色，强度零点五五，提供基底漫反射，消除大面积死黑。第二层是半球光，上方暖金模拟洞窟顶部光源，下方暖灰棕模拟地面漫反射。第三层是定向主光，暖金色从斜前方打亮佛像正面，附带柔和阴影。第四层是点光源，每间洞窟三个核心光源，造像主光从前方打亮佛龛，轮廓勾边光从后方侧打分离造型与背景，壁画补光居中照射壁画区域。第五层是走廊辅光，在关键转角处设置侧光引导参观动线。第六层是焦点聚光，仅对盛唐四十五窟主尊额外加强，作为全馆第一视觉焦点。

整个光照系统的设计原则是保留暗光展陈氛围，同时确保展品细节清晰可辨。

### 2.5 PBR 材质与实例合并

材质方面，我定义了九种预设，分别是壁画、彩塑、墙体、地板、画框、藻井、崖壁、栈道和装饰线。每种预设都有独立的粗糙度、金属度、凹凸强度、环境贴图强度和自发光参数。

壁画材质粗糙度零点八五，无金属度，微弱自发光避免死黑。彩塑材质粗糙度零点五，金属度零点三，提升暗部反光还原造型结构。墙体粗糙度零点九，哑光石材质感。画框金属度零点二，哑光暗金不刺眼。

同一预设类型的所有 Mesh 共享一个材质实例，这是 WebGL 的优化手段。材质实例合并后，绘制批次从百余次降至三十以下，显著降低 GPU 开销。

### 2.6 低模空间结构

展品模型在 Blender 中精细建模后导入，但墙体、画框、基座、标题这些空间结构体不需要高模细节。我在 Three.js 中用代码实时生成这些低模结构，包括洞窟后墙、右墙、顶棚、壁画框、造像基座、走廊地面和导览墙。左侧隔断墙全部删除，保证漫游时视野开阔不被遮挡。每间洞窟的标题用 Canvas 绘制文字纹理，以 Sprite 形式悬浮在洞窟后墙上方。

这种高低模混合策略既保证了展品的精细度，又控制了整体面数，还降低了模型文件体积。

### 2.7 视角控制与交互

视角控制融合了三种方式。鼠标拖动做轨道环绕观赏，这是 OrbitControls 的标准功能。键盘 W 前进、S 后退、A 左移、D 右移做自由漫游，为了消除顿挫感，我实现了平滑缓动插值和减速衰减，松开按键后不是立即停止而是缓慢减速。滚轮支持多维环绕，普通滚轮水平旋转视角，Shift 加滚轮垂直仰俯视角，Ctrl 加滚轮做缩放。

顶部有五个洞窟的导航按钮，点击任意一个，镜头会平滑飞行到对应洞窟位置，飞行过程有缓动插值，不是瞬间跳切。飞行到达后自动切换洞窟标题的可见性。

展品交互通过 Raycaster 射线拾取实现。点击壁画或彩塑，射线与场景中的物体求交，根据交点物体的名称匹配解说数据库。我收录了三十多条文物解说词条，按长键优先策略匹配，避免短键误判。比如"285_左墙_飞天"这条长键会优先于"飞天"这条短键匹配，确保解说内容精确对应到具体洞窟的具体位置。

右下角有缩略导览地图，实时显示相机在展馆中的位置，用脉冲标记跟随移动，迷路时看一眼就知道自己在哪。

### 2.8 三重粒子特效

空中有三种粒子在飘动。花瓣粒子两百个，模拟敦煌壁画中常见的飞天散花场景。光晕粒子六十个，在藻井和佛龛附近营造微弱的光芒氛围。金沙粒子一百五十个，模拟戈壁洞窟中浮尘在光束中闪烁的视觉效果。

金沙粒子按三个区域分配，镇馆文物附近占百分之五十密度最高，藻井区域占百分之三十，环境氛围占百分之二十，这样视觉焦点自然集中在展品上。

三种粒子都使用程序化 Canvas 纹理生成外观，花瓣是粉色椭圆带透明渐变，光晕是径向渐变圆点，金沙是小方形亮金点。粒子更新频率降为一帧一次而不是每帧更新，减少 GPU 开销。

### 2.9 遇到的困难与克服

第一个困难是模型体积。建完的 GLB 文件体积偏大，网页加载要等很久。解决方式是 Meshopt 压缩配合 Service Worker 缓存，首次加载压缩后体积可控，二次访问直接从缓存读取秒开。

第二个困难是光照死黑。最初只用了环境光和一盏定向光，洞窟深处完全漆黑，壁画和彩塑的细节全部丢失。我经历了多轮调试，从两层光照逐步增加到六层，每一层解决一个具体问题，最终在保留博物馆暗光氛围的同时让展品清晰可辨。

第三个困难是 Draw Call 过高。每个 Mesh 用独立材质实例时，绘制批次超过一百，帧率明显下降。通过材质实例合并，相同预设类型的 Mesh 共享材质，批次降到三十以下。

第四个困难是 WASD 漫游顿挫。按住方向键时镜头一跳一跳地移动，松开时突然停止，体验很差。实现平滑缓动插值和减速衰减后，移动变得流畅自然。

第五个困难是白屏崩溃。某次推送代码后，打开网页直接白屏什么都不显示。查了很久发现是 caveStructure.js 中调用建窟函数时多传了一个 leftWallMaterial 参数，而这个变量从未定义。删掉这一个参数，画面恢复正常。

第六个困难是 GitHub Pages 子路径部署。项目部署在 dunhuang-3dshow 子路径下，很多资源的绝对路径都指向根路径，导致 404。把所有路径改成相对路径，Service Worker 的作用域动态适配，部署终于稳定。

第七个困难是 Service Worker 缓存版本。更新代码后推送到 GitHub，但浏览器还在用旧缓存，看到的一直是旧版本。在 Service Worker 中加入版本号机制，代码更新时版本号递增，旧缓存自动失效。

## 三、展馆外貌与设计寓意

展馆整体是一个七十二米长、十六米宽的矩形空间，外观取意于莫高窟九层楼的横向展开形态，但简化为纯粹的功能性空间，不做过度的外观装饰，将视觉注意力留给内部的洞窟和展品。

五间洞窟沿 X 轴中轴线从左到右按朝代排列，西魏在最左端，元代在最右端。这个排列顺序本身就是一条时间线，游客从左走到右，就是从公元六世纪走到十四世纪，穿越一千年的营建史。中轴线走廊宽三点五米，地面使用浅棕拉丝石材材质，模拟莫高窟洞窟内的石板地面。

左侧隔断墙全部删除，这是经过反复权衡的决定。保留隔断墙更有围合感，但会严重遮挡壁画和造像。最终选择开放左侧，用暗光氛围本身来营造空间感，而不是依赖墙体围合。右侧保留一堵低矮导览墙，高度仅一米，透明度零点六，弱化视觉存在感的同时引导游客沿中轴线行进。

每间洞窟的入口处有暗金色画框，取意于博物馆展陈中的画框语汇，提示游客即将进入一个独立的展陈空间。洞窟后墙上方悬浮着金色标题 Sprite，显示朝代和窟号，进入洞窟时自动显示，离开时自动隐藏。

展馆尽头有一处观景平台，游客走完五间洞窟后可以在这里回头远眺整条中轴线，金沙粒子在光束中缓缓飘散，像极了莫高窟崖面上夕阳西照时的感觉。

## 四、五间洞窟各窟特色与文化内涵

### 西魏 285 窟

第285窟是莫高窟最早有明确纪年的洞窟，开凿于西魏大统四年，即公元538年。这间洞窟的文化价值在于它是中西文化交融的见证。

左墙绘有飞天形象，融合了中国传统神话中的伏羲女娲题材。飞天面容清瘦秀朗，飘带飞扬，与后来盛唐的丰腴飞天截然不同，体现了从西域粗犷风格向中原清秀风格的转变，是敦煌飞天演变的关键节点。

右墙绘有天宫伎乐，乐伎手持琵琶、箜篌、笛子等乐器，身姿优美。这些乐器形制融合了中原与西域的特点，是研究古代音乐史的重要图像资料。

后墙绘有五百强盗成佛图，是敦煌最早的因缘经变画之一。画面讲述五百强盗被王军征讨、受刑、皈依佛法最终成佛的故事，构图宏大，情节连贯，体现了佛教"放下屠刀，立地成佛"的教义。

窟顶藻井以莲花为中心，四周环绕飞天与卷草纹，色彩以石青、石绿为主调，融合了中原与西域的装饰风格。

佛龛内供奉西魏禅定佛，承袭南朝"瘦骨清像"风格。面容清瘦秀朗，身体修长挺拔，衣纹繁复贴体如出水之态，结跏趺坐于简洁石质方座之上，施禅定印，神态超然脱俗。两侧胁侍菩萨同样清瘦修长，石青色飘带贴体飞扬，与壁画中的飞天一脉相承。这组造像以清逸飘举的线条取胜，与盛唐的丰腴华贵形成鲜明对比。

### 盛唐 45 窟

第45窟是全馆的第一视觉焦点，开凿于八世纪盛唐时期。这间洞窟代表了敦煌彩塑与壁画艺术的巅峰水平。

左墙的盛唐飞天面相丰腴，体态婀娜，飘带华丽流畅。盛唐飞天是敦煌飞天艺术的最高成就，它们不长翅膀、不生羽毛，仅凭飘带即可凌空飞舞，这是中国艺术家独特的创造。

右墙绘有天宫伎乐，盛唐乐伎形象丰腴优美，乐器形制与唐代宫廷音乐制度相符，反映了丝绸之路上东西方音乐文化的深度交流。

后墙绘有观无量寿经变，描绘阿弥陀佛西方净土的宏大场景，画面中央为阿弥陀佛说法，两侧观音、大势至菩萨胁侍，周围环绕圣众、楼阁、宝池、莲花，构图严谨，色彩绚丽。

佛龛内的彩塑群是这间洞窟的核心。主佛面相丰腴庄严，左侧迦叶老成持重，右侧阿难年轻聪慧，外侧菩萨身披璎珞婀娜优美，最外侧天王身披铠甲威武守护。七尊造像性格各异，是盛唐彩塑的代表作。

我在光照上对这间洞窟做了额外加强，主尊顶部增设一盏焦点聚光，让游客从走廊就能看到佛龛中的金光，被吸引走入洞窟。

### 盛唐 217 窟

第217窟开凿于八世纪初，以法华经变和观无量寿经变闻名于世。

后墙绘有法华经变化城喻品，这是敦煌最著名的经变画之一。画面描绘山峦叠嶂、楼阁亭台的佛国净土景象，以青绿山水为主调。化城喻品讲述众人寻宝途中疲惫欲退，导师化现城池供其休息后继续前行的故事，寓意修行需要精进不懈。这幅画开创了中式青绿山水的全新范式，对后世山水画影响深远。

窟顶藻井绘有团花图案，以莲花纹样为中心八瓣对称展开，外围搭配卷草纹、联珠纹环绕，色彩饱满浓烈，交替使用石青、石绿、朱砂矿物颜料，代表盛唐洞窟装饰艺术的最高水准。

佛龛内供奉盛唐主佛，结跏趺坐于双层束腰须弥座之上，面相丰腴庄严，衣纹流畅。两侧胁侍菩萨头戴宝冠，身披璎珞，体态婀娜，石青色调华贵醒目。外侧供养菩萨体量较小，形成由高到低三层纵深布局，体现了盛唐洞窟彩塑严整的等级秩序。

### 晚唐 17 窟（藏经洞）

第17窟是莫高窟最负盛名的洞窟，即藏经洞。这间洞窟原本是晚唐高僧洪辩法师的影窟，内塑洪辩真容像。公元1900年，王道士在清扫积沙时偶然发现了这间洞窟中封存的密室，里面藏有五万余件从四世纪到十一世纪的文献、绢画、经卷等文物，时间跨度近千年。

藏经洞的发现是二十世纪中国最重大的考古发现之一，直接催生了敦煌学这一国际显学。但与此同时，大量文物被斯坦因、伯希和等人运往海外，散落于大英博物馆、法国国家图书馆等机构，这是中国近代史上的文化伤痛。

在我的数字洞窟中，17窟展示藏经洞出土的绢画，包括佛说法图、菩萨像、经变画等，色彩以矿物颜料绘制，历经千年仍鲜艳如初。甬道侧壁绘制晚唐僧人画像，完整还原当时敦煌佛教僧团的真实样貌。

佛龛内供奉洪辩法师真容像，结跏趺坐于经卷展台之上，面容庄严写实，身着袈裟，体现了晚唐彩塑由盛唐的理想化向写实化转变的风格。两侧弟子像一老一少，老者沧桑持重，少者恭顺聪慧。前方小型僧人像体量更小，形成尊卑有序的纵深层次，生动还原九世纪敦煌僧团的层级结构。

藻井延续了盛唐华丽纹样体系，但整体审美转向含蓄内敛，以土黄、赭石为基底，少量石青点缀，体现了晚唐独有的素雅风格，也折射出大唐帝国由盛转衰的时代情绪。

### 元代 3 窟

第3窟是莫高窟最晚期的代表洞窟之一，开凿于十四世纪元代，也是敦煌仅存的完整元代石窟。这间洞窟标志着莫高窟千年营建的终结。

后墙绘有千手千眼观音经变，是这间洞窟最具代表性的文物。千手观音象征法力无边、普度众生，壁画采用铁线描技法，线条精细流畅，色彩以石绿、朱砂为主。

左墙绘有密宗千佛图，以密宗诸尊替代传统千佛，排列整齐的密宗尊像象征密宗曼荼罗宇宙观。右墙绘有密宗胁侍菩萨，头戴五佛冠，手持法器，融合了藏传佛教的造型特点，体现了元代汉藏佛教艺术的交融。

供养壁画中出现了蒙古、汉、西域多族群形象，服饰各有区分，直观展现元代河西多民族文化交融的社会面貌。

佛龛内供奉千手千眼观音，立于密宗坛城基座之上，是全馆唯一的密宗主尊造像。观音多臂呈放射状展开，石绿色调，视觉冲击力极强，与西魏的清瘦、盛唐的丰腴截然不同。两侧胁侍菩萨头戴五佛冠，高低错落，融合藏传佛教造型特点。最外侧护法金刚身披朱砂色铠甲，怒目威猛，与盛唐天王力士的武将风格不同，更强调密宗的神秘威严。整组造像从千手观音到胁侍菩萨再到护法金刚，构成密宗坛城的三层纵深布局。

窟顶藻井以密宗曼荼罗图案为核心，融合藏传佛教元素与中原装饰传统，为敦煌千年藻井艺术画上了最后的句号。

从西魏285窟到元代3窟，五间洞窟串起了一条完整的千年营建史。每间洞窟的风格都忠实于它所属的时代，西魏清秀飘逸，盛唐丰腴华丽，晚唐含蓄内敛，元代刚劲密丽。游客从左走到右，就是在穿越敦煌一千年的艺术演变。

## 五、总结

这个项目从一根线开始，用代码重建了一座展馆。三千四百行 Blender Python 脚本程序化建模，六层分层光照反复调试消除死黑，九种 PBR 材质预设统一敦煌矿物配色，材质实例合并降低绘制批次，低模空间结构实时生成控制面数，Raycaster 射线拾取三十余条文物解说，三重粒子特效四百一十个粒子营造沉浸氛围，平滑缓动 WASD 漫游消除顿挫，五洞窟飞行跳转缩略地图实时定位，Service Worker 缓存秒开，GitHub Actions 自动部署。

过程中攻克了白屏崩溃、光照死黑、模型体积、部署路径、缓存版本等一系列技术难题。最终实现的效果是，打开一个网址，就能独自漫游五间跨越千年的敦煌洞窟，看壁画、看彩塑、听解说，想看多久就看多久。

一千六百年的色彩，不该只留在洞窟里。
